This dashboard is read-only. It traces the four-tier price resolver cascade — Chainlink, Pyth, CoinGecko, DexScreener — and shows exactly which source provided the accepted price for each of the 500 registry assets.

Production resolver behavior is unchanged. This page only visualises its output. Use it to verify CoinGecko IDs, confirm DexScreener fallback coverage, and identify any remaining assets without a price source.

Data is refreshed by running node generate-qa-data.js which calls the resolver and writes price-verification-data.json.

Four tiers.
In order, always.
First success stops the chain.

01
Tier 1
Chainlink
On-chain ETH mainnet push feeds. Requires CHAINLINK_RPC_URL. Highest trust. If a valid price is returned, tiers 2–4 are never called.
02
Tier 2
Pyth
Off-chain pull feeds via Hermes REST. Chunked at 40 IDs per request to avoid URL length limits. ~117 feeds configured.
03
Tier 3
CoinGecko
Single bulk REST call for all IDs. Free tier supported. CoinGecko + DexScreener together are the complete verification standard — Chainlink feeds are never required.
04
Tier 4
DexScreener
Full independent fallback for every token. No API key required. Selects highest-liquidity pair by USD liquidity. Covers all chains including Base, Solana, BNB, ETH.
